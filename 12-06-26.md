### Documentación técnica paso a paso

> **“MAVLink es un protocolo ligero de mensajería diseñado para la comunicación entre drones y estaciones de control en tierra. Es eficiente, confiable y ampliamente usado en sistemas de vehículos aéreos no tripulados (UAV), permitiendo transmitir telemetría, comandos y datos entre múltiples dispositivos.”**

## 1. Requisitos previos y recomendaciones generales
- **Sistema recomendado:** Ubuntu 22.04 (nativo) o WSL2 con GUI bien configurada.  
- **Recursos mínimos:** CPU moderna, 8+ GB RAM (16 GB recomendado para Gazebo), GPU discreta para simulación 3D compleja.  
- **Red y puertos:** SITL publica MAVLink por UDP en **14550** (por defecto). Permite ese puerto en firewall.  
- **Versiones:** usa la **última release estable** de PX4 y una distribución LTS de ROS 2 (ej. Humble o la LTS actual).  
- **Seguridad:** nunca pruebes con hélices puestas en hardware durante calibraciones iniciales o HITL.

---

## 2. MAVLink — qué es y cómo usarlo
**Qué es:** protocolo ligero de mensajería para telemetría y comandos entre autopilotos, simuladores y estaciones de control.  
**Uso práctico:** QGroundControl, PX4 y simuladores intercambian mensajes MAVLink; los scripts y nodos ROS usan `pymavlink` o `mavros` para comunicarse.

### Instalación de librerías (ejemplos)
- **Python (pymavlink):**
```bash
python3 -m pip install --user pymavlink
```
- **Generar bindings C/C++ desde XML (mavgen):**
  - Usar `mavgen` incluido en herramientas MAVLink o generar con `mavlink` repo según documentación oficial.

### Ejemplo de conexión simple (Python)
```python
from pymavlink import mavutil
master = mavutil.mavlink_connection('udp:127.0.0.1:14550')
master.wait_heartbeat()
print("Heartbeat from system (system %u component %u)" % (master.target_system, master.target_component))
```

## 3. PX4 (firmware) — instalación y uso en SITL
**Qué es:** firmware open‑source que corre en Pixhawk y en modo SITL para pruebas en PC.

### Instalación básica (Ubuntu)
```bash
# Instalar dependencias (script oficial)
git clone --recursive https://github.com/PX4/PX4-Autopilot.git
bash ./PX4-Autopilot/Tools/setup/ubuntu.sh
# Abrir nueva terminal o recargar perfil
. ~/.profile
```

### Compilar y ejecutar SITL
- **Con jMAVSim (rápido):**
```bash
cd PX4-Autopilot
make px4_sitl_default jmavsim
```
- **Con Gazebo (alta fidelidad):**
```bash
cd PX4-Autopilot
make px4_sitl_default gazebo
```

**Salida esperada:** PX4 arranca en modo SITL y expone MAVLink por UDP (puerto 14550). jMAVSim o Gazebo muestran la escena.

## 4. jMAVSim — simulador ligero (instalación y uso)
**Qué es:** simulador Java ligero para PX4, ideal para pruebas rápidas en máquinas con pocos recursos.

### Requisitos
- **JDK 17** instalado:
```bash
sudo apt update
sudo apt install openjdk-17-jdk
```

### Ejecutar jMAVSim con PX4
```bash
cd PX4-Autopilot
make px4_sitl jmavsim
```
- **Control:** conecta QGroundControl a `localhost:14550` o usa `pymavlink`/MAVROS.
- **Configuración avanzada:** parámetros de vehículo en `ROMFS/px4fmu_common` o archivos de configuración del repositorio PX4.

---

## 5. Gazebo — simulador avanzado (instalación y uso)
**Qué es:** simulador 3D con motores de física y sensores simulados (cámaras, LiDAR, IMU).

### Instalación (Ubuntu + ROS 2 Humble ejemplo)
```bash
sudo apt update
sudo apt install ros-humble-gazebo-ros-pkgs gz-sim
```

### Ejecutar PX4 + Gazebo
```bash
cd PX4-Autopilot
make px4_sitl_default gazebo
# o para un modelo específico
make px4_sitl gazebo_iris
```

### Integración con ROS 2
- Instala **MAVROS** o paquetes de puente PX4↔ROS2.
- Lanzar nodo MAVROS (ROS 2):
```bash
ros2 run mavros mavros_node
```
- Usa topics y servicios ROS 2 para leer sensores y enviar comandos.

## 6. ROS 2 — integración con drones (MAVROS)
**Qué es:** middleware para nodos, topics, servicios y acciones; se usa para lógica de alto nivel, planificación y procesamiento de sensores.

### Instalación básica (Humble ejemplo)
```bash
sudo apt update
sudo apt install ros-humble-desktop
```

### Integración PX4 ↔ ROS 2
- **MAVROS** (ROS 2) actúa como puente entre MAVLink y ROS topics.
- Ejemplo de uso: publicar comandos de posición desde un nodo ROS 2 hacia el dron simulado.

---

## 7. QGroundControl — estación de control (GCS)
**Qué es:** GCS multiplataforma para planificar misiones, ver telemetría y flashear firmware.

### Instalación
- Descargar la app desde la web oficial (Windows/macOS/Linux/Android/iOS) y ejecutar el instalador correspondiente.

### Conexión a SITL
- SITL (PX4) publica MAVLink en `udp:127.0.0.1:14550`. QGroundControl detecta la conexión automáticamente.
- Si no detecta, en QGC: **Application Settings → Comm Links → Add → UDP → Host: 127.0.0.1 Port: 14550**.

### Flasheo de Pixhawk
- Conecta Pixhawk por USB; en QGC ir a **Firmware** → seleccionar PX4 → seguir pasos de flasheo.

---

## 8. Pixhawk 6X — hardware y HITL
**Qué es:** controlador FMUv6X con múltiples IMUs, diseñado para vuelo real y pruebas HITL.

### Preparación y flasheo
1. Conectar Pixhawk 6X por USB al PC.  
2. Abrir QGroundControl → **Firmware** → seleccionar PX4 → flashear.  
3. Realizar **calibraciones**: acelerómetro, brújula, radio, ESCs (si aplica).  
4. Para **HITL**: configurar PX4 para usar puerto serial/UDP que conecta al simulador; seguir guía PX4 para modo HITL.

### Precauciones
- **Quitar hélices** durante calibraciones y pruebas en banco.  
- Verificar parámetros de failsafe y límites de arming antes de vuelo real.

---

## 9. Flujo de trabajo recomendado (SITL → HITL → vuelo real)
1. **SITL + jMAVSim**: pruebas rápidas de control y lógica.  
2. **SITL + Gazebo**: pruebas de sensores, visión y navegación avanzada.  
3. **ROS 2 + MAVROS**: integrar algoritmos de alto nivel (planificación, SLAM, percepción).  
4. **HITL con Pixhawk 6X**: validar controladores y latencias con hardware real sin hélices.  
5. **Vuelo real**: después de validaciones y pruebas de seguridad.

## 11. Problemas comunes y soluciones rápidas
- **QGC no detecta SITL:** comprobar que SITL está publicando en `127.0.0.1:14550`; desactivar firewall o permitir puerto UDP.  
- **jMAVSim no arranca:** verificar JDK 17 y variables `JAVA_HOME`.  
- **Gazebo lento o crasheos:** reducir calidad gráfica o usar máquina con GPU; comprobar versiones compatibles PX4↔Gazebo.  
- **Errores de MAVROS:** revisar que el nodo MAVROS use el mismo puerto/baud que SITL/HITL.

## 12. Buenas prácticas y recomendaciones finales

- **Pruebas incrementales:** valida cada capa (SITL → simulador → GCS → HITL) antes de avanzar.  
- **Documentación oficial:** consulta las guías oficiales de PX4, MAVLink, Gazebo y ROS 2 para detalles avanzados y compatibilidades.
