## 1. Instalación de Herramientas de Conexión Remota (SSH)
```c
sudo apt-get install ssh
```
Instala el paquete completo de SSH. Incluye el cliente (para conectarte a otros equipos) y el servidor (para recibir conexiones).

```c
sudo apt install openssh-server
```
Instala específicamente el componente de servidor. Permite que tu máquina acepte conexiones remotas de forma segura.

## 2. Gestión del Servicio SSH
```c
sudo systemctl start ssh
```
Enciende el servidor SSH en este momento. Activa la escucha de conexiones entrantes de inmediato.

```c
sudo systemctl enable ssh
```
Configura el servidor SSH para que se inicie automáticamente cada vez que la computadora se encienda.

## 3. Red y Diagnóstico (Net-Tools)
```c
sudo apt install net-tools
```
Instala un conjunto clásico de herramientas de red para Linux. Incluye comandos antiguos pero muy usados como ifconfig, netstat y route.

```c
sudo ifconfig
```
Muestra la configuración de red actual de tu equipo. Te permite ver la dirección IP local (en la línea que dice inet), la máscara de subred y las interfaces activas.

## 4. Escaneo de Seguridad (Nmap)
```c
sudo apt install nmap
```
Instala el escáner de redes Nmap. Sirve para descubrir qué dispositivos están conectados a una red y qué puertos tienen abiertos.

```c
sudo nmap [IP de inet]
```
Analiza tu propia computadora utilizando su dirección IP. Te muestra qué puertos y servicios (como el SSH que acabas de activar) están abiertos y expuestos en la red.

## 1. El formato correcto para usuarios de dominio (En el PC 1)
Cuando usas cuentas de dominio, no basta con poner el nombre del usuario a secas. Debes indicarle explícitamente el dominio a SSH.
Prueba conectarte desde el PC 1 usando uno de estos dos formatos:

* Formato con barra invertida (Debe llevar triple barra en WSL/PowerShell para que no se rompa el comando):
```C
ssh DOMINIO\\\usuario_pc2@IP_del_pc2
```
Ejemplo: `ssh MIEMPRESA\\\juan@192.168.1.15` 
* Formato alternativo (Estilo correo):
```C
ssh usuario_pc2@dominio.local@IP_del_pc2
```
Ejemplo: `ssh juan@miempresa.local@192.168.1.15} 

## 2. Permitir el grupo del dominio en la configuración SSH (En el PC 2)
Por defecto, OpenSSH en Windows solo deja pasar a administradores locales. Tienes que indicarle que acepte a los usuarios de tu dominio corporativo. 

   1. En el PC 2, abre el bloc de notas como Administrador y abre este archivo: C:\ProgramData\ssh\sshd_config
   2. Ve hasta el final del archivo y busca si existen líneas que digan AllowUsers o AllowGroups.
   3. Agrega la siguiente línea al final de todo para autorizar a tu grupo del dominio (reemplaza por tus datos):
   
   AllowGroups "DOMINIO\Usuarios del dominio" Administrators
   
   4. Guarda el archivo y reinicia el servicio desde PowerShell como Administrador:
   
   Restart-Service sshd

# SSH
**El protocolo Secure Shell (SSH)** es un método para enviar comandos de forma segura a un ordenador a través de una red no segura. 
- SSH utiliza criptografía para autenticar y cifrar las conexiones entre dispositivos.
- SSH también permite la tunelización o el reenvío de puertos, lo que permite que los paquetes de datos atraviesen redes que de otro modo no podrían cruzar.
- SSH se utiliza frecuentemente para controlar servidores de forma remota, gestionar infraestructuras y transferir archivos.

**Telnet** es el predecesor de SSH. Podríamos decir que es algo así como http a https. 
- Mediante telnet puedes conectarte a un equipo remoto y mover archivos de una carpeta a otra o eliminarlos, es decir, usar los comandos típicos de SSH (como cp o rm) o realizar tareas más complejas como la creación de una base de datos e inserción de contenido en la misma.
- La gran desventaja de telnet es que las conexiones son en texto plano. Esto significa que un usuario que espía tus conexiones de red podrá entender toda la conversación entre el servidor y tu ordenador. No solo eso: podrá capturar la contraseña de forma sencilla y colarse en el servidor.

En su forma más básica, SSH te permitirá conectarte a un equipo remoto mediante un usuario y contraseña para que puedas comenzar a explotar todo el potencial del servidor a tu disposición con un par de comandos.

- **Servidor SSH:** Es el programa que configuras en el servidor y que acepta las conexiones SSH.

- **Cliente SSH:** Es la aplicación con la que realizas una conexión SSH y que, en términos informáticos, actúa a modo de cliente. Dependiendo del sistema operativo que tengas, podrías tener un cliente SSH instalado por defecto o bien necesitarás instalar uno.

Usar SSH en WSL generalmente se divide en dos casos: conectarte desde WSL a un servidor remoto (lo más común) o acceder a tu WSL desde otro equipo o desde Windows.

---

## 1. Conectarte desde WSL a un servidor remoto (Cliente)

  WSL ya trae el cliente SSH preinstalado. Ábrelo y conéctate como en cualquier máquina Linux:
  `ssh usuario@direccion_ip_o_dominio`
   Para añadir tus claves criptográficas (por ejemplo, para GitHub):
   1. Genera una clave con `ssh-keygen`.
   2. Copia tu llave pública y agrégala a tu servidor.
      
## 2. Conectarte a tu WSL desde Windows (Servidor)
   
  Para acceder a tu terminal de WSL desde el símbolo del sistema (CMD) o PowerShell en tu misma PC:
  1. Dentro de tu distribución WSL, abre el archivo de configuración con `sudo nano /etc/ssh/sshd_config`.
  2. Asegúrate de que las siguientes líneas estén descomentadas (sin el # al inicio) y configuradas así:
  ```c
  textPort 2222
  ListenAddress 0.0.0.0
  PasswordAuthentication yes
  ```
  3. Inicia o reinicia el servicio SSH: `sudo service ssh restart` o `sudo systemctl restart ssh`.
  4. En tu ventana de PowerShell de Windows, ingresa:
  ```c 
  powershell
  ssh tu_usuario_de_wsl@localhost -p 2222
  ```
## 3. Conectarte desde otra computadora a tu WSL

  Debido a la arquitectura de red de WSL (que actúa como una máquina virtual), para acceder desde otro equipo necesitas redirigir el tráfico del puerto de tu PC a la instancia de WSL.
  1. Configura el servidor SSH dentro de tu WSL (pasos 1 y 2 de la sección anterior).
  2. Obtén la IP interna de WSL ejecutando `ip addr | grep eth0` dentro de tu WSL y anotando la dirección inet.
  3. En Windows, ejecuta PowerShell como Administrador y crea una regla de reenvío de puertos para que Windows escuche el tráfico y lo redirija a WSL.
  ```c 
  powershell
  netsh interface portproxy add v4tov4 listenport=2222 listenaddress=0.0.0.0 connectport=2222 connectaddress=<IP_DE_WSL_QUE_ANOTASTE>
  ```
## 4. Conéctate desde el otro equipo usando la dirección IP de tu computadora Windows y el puerto redirigido:
  ```c 
  bash
  ssh tu_usuario_de_wsl@IP_DE_TU_PC_WINDOWS -p 2222
  ```

# Crear directorios en un servidor remoto mediante SSH

Conéctate desde tu WSL al servidor remoto usando `ssh usuario@direccion_ip`, luego navega con `cd` y crea tu carpeta con `mkdir nombre_carpeta_remota`. Para rutas complejas usa `mkdir -p`.

Si necesitas crear un directorio dentro de otra computadora o servidor usando SSH desde tu terminal WSL, primero conéctate al servidor y luego ejecuta el comando de creación.
Conéctate a tu servidor remoto:
``` c
bash 
ssh usuario@direccion_ip
``` 
Una vez dentro del servidor, crea la carpeta deseada:
```c 
bash 
mkdir nombre_carpeta_remota
```
